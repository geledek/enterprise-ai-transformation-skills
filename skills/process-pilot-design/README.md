# Pilot Design

A seven-role structured protocol for designing a 90-day AI pilot with pre-deployment metrics, stop conditions, and workflow redesign.

## What it does

Produces a one-page pilot brief ready for sponsor approval, covering:
- Scope classification using the NANDA Scope × Execution Grid
- Pre-deployment metrics with measurement-gap audit
- Blast-radius assessment and reliability bar
- Stop conditions at day 30, 60, and 90
- Workflow redesign before technology deployment
- Named ownership (owner, sponsor, user champion)

## When to use it

After an AI idea has been gated (use `general-idea-diagnostic` first). Before any technology procurement, vendor evaluation, or developer sprint. Any time someone says "let's pilot this."

## What it outputs

A one-page pilot brief: problem statement, scope, redesigned workflow, 3 pre-deployment metrics, blast radius, stop conditions, ownership, 90-day timeline.
